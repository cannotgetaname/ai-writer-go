# 架构师页面 UI 重构设计

## 背景

当前架构师页面存在以下问题：
1. **信息密度过高** - 左右两栏挤满了表单和数据，视觉压力太大
2. **操作反馈弱** - 生成状态、进度、成功/失败反馈不够直观
3. **排版混乱** - 左右两栏不平衡、步骤条和内容脱节、卡片堆叠无分组、间距问题

## 设计目标

- 保持双栏布局，但重新组织内容和视觉层次
- 左侧改为步骤导航（卡片式），右侧为工作区
- 支持自由切换步骤，尤其是章节大纲阶段
- 所有 AI 生成结果用户均可编辑和保存
- 前置步骤修改后，后续步骤标记"可能过时"但不删除

## 组件设计

### 1. 左侧导航卡片

**卡片结构：**
```
┌─────────────────────────┐
│ ① 全书总纲              │  ← 序号 + 名称
│ ─────────────────────── │
│ ✓ 已完成                │  ← 状态标签
│ 《仙途问道》            │  ← 摘要第一行
│ 玄幻 · 100万字          │  ← 摘要第二行
└─────────────────────────┘
```

**状态定义：**

| 状态 | 图标 | 标签颜色 | 含义 |
|------|------|---------|------|
| 未开始 | ○ | 灰色 `#909399` | 还未生成数据 |
| 当前 | ● | 蓝色 `#0071e3` | 正在处理的步骤 |
| 已完成 | ✓ | 绿色 `#67c23a` | 数据已生成并保存 |
| 可能过时 | ⚠️ | 橙色 `#e6a23c` | 前置步骤已修改，建议重新生成 |
| 生成中 | ⟳ | 蓝色动画 | AI正在生成 |

**交互规则：**
- 点击卡片切换到该步骤，右侧工作区更新
- 当前步骤卡片有蓝色左边框高亮（4px solid #0071e3）
- 非当前步骤 hover 时背景色变化（#f5f7fa）
- 每个步骤卡片高度约 100px，内容垂直居中

**各步骤摘要内容：**

| 步骤 | 摘要第一行 | 摘要第二行 |
|------|-----------|-----------|
| 1. 全书总纲 | 书名 | 题材 · 预估字数 |
| 2. 世界观 | 时代背景 | 力量体系（截断） |
| 3. 分卷大纲 | 共 X 卷 | 已展开 X/Y 卷 |
| 4. 章节大纲 | 已展开 X/Y 卷 | 共 Z 章 |
| 5. 章节细纲 | 已生成 X 章 | - |

**上下文预览：**
- 显示前置步骤的关键信息（一行摘要）
- 点击展开查看完整内容
- 帮助用户理解当前步骤的输入依据

**结果区编辑模式：**
- 所有字段默认为输入框/文本域（内联编辑，无需弹窗）
- 长文本字段使用 textarea（rows=3）
- 短文本使用 input

- 底部固定"保存"和"重新生成"按钮
- 编辑后自动进入"未保存"状态，显示保存提示

## 过时数据逻辑

### 触发条件

当某步骤的数据被修改（重新生成或手动编辑保存），其后续所有步骤标记为"可能过时"。

**依赖关系：**
```
总纲 → 世界观 → 分卷大纲 → 章节大纲 → 章节细纲
```

**示例：**
- 修改总纲 → 世界观、分卷大纲、章节大纲、章节细纲都标记过时
- 修改世界观 → 分卷大纲、章节大纲、章节细纲标记过时
- 修改分卷大纲 → 章节大纲、章节细纲标记过时
- 展开新分卷的章节 → 不影响其他步骤（是增量操作）

### 状态计算

```javascript
// 过时状态存储
const outdatedSteps = ref(new Set()) // 存储过时的步骤索引

// 标记后续步骤过时
function markSubsequentStepsOutdated(stepIndex) {
  for (let i = stepIndex + 1; i < 5; i++) {
    outdatedSteps.value.add(i)
  }
}

// 检查步骤状态
function getStepStatus(stepIndex) {
  if (loading && currentStep === stepIndex) return 'generating'
  if (outdatedSteps.value.has(stepIndex)) return 'outdated'
  if (stepIndex === currentStep) return 'current'
  if (hasData(stepIndex)) return 'completed'
  return 'pending'
}
```

## 数据流

### 数据结构

```javascript
// 步骤数据
const stepData = {
  synopsis: null,           // 总纲
  worldView: null,          // 世界观
  volumes: [],              // 分卷大纲
  chapterDetails: {}        // 章节细纲 (key: "卷索引_章索引")
}

// 当前步骤 (0-4)
const currentStep = ref(0)

// 过时标记
const outdatedSteps = ref(new Set())

// 分卷展开状态（用于折叠/展开UI，以及跟踪哪些卷已生成章节）
const volumeExpanded = ref({}) // { 0: true, 1: false, ... } - true表示该卷已展开章节
```

### 保存逻辑

1. 每次操作（生成/编辑保存）后调用 `saveArchitectData()`
2. 保存时一并保存 `outdatedSteps` 状态
3. 加载时恢复 `outdatedSteps`，并重新计算（基于数据版本）

### 数据版本追踪

使用简单的依赖记录，而非版本号：

```javascript
// 保存时记录依赖
function saveWithDependency(stepKey, dependsOn = null) {
  const data = {
    [stepKey]: stepData[stepKey],
    dependencies: dependsOn ? { [stepKey]: dependsOn } : {}
  }
  // 合并到现有依赖记录
  dependencies.value = { ...dependencies.value, ...data.dependencies }
  saveArchitectData()
}

// 检查过时：比较前置步骤的修改时间
function checkOutdated() {
  // 如果 synopsis 更新时间 > worldView 创建时间，则 worldView 过时
  if (synopsisResult.value?.updatedAt > worldViewResult.value?.createdAt) {
    outdatedSteps.value.add(1) // 世界观过时
  }
  // ... 其他步骤类似
}
```

数据结构中添加时间戳：
```javascript
{
  synopsis: { ...data, updatedAt: timestamp },
  worldView: { ...data, createdAt: timestamp, updatedAt: timestamp }
}
```

## 风险与注意事项

1. **数据迁移**：现有用户的架构师数据需要兼容新格式
2. **性能**：章节大纲步骤可能有大量数据，需注意渲染性能
3. **响应式适配**：
   - 当前设计针对桌面端（宽度 ≥ 1024px）
   - 平板端（768-1024px）：左侧导航收窄至 240px
   - 移动端（< 768px）：左侧导航隐藏，改为顶部步骤条 + 下拉选择
